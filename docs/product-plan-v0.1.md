# 羽毛球组局小程序 · 产品粗粒度方案 v0.1

> 工作名：**来打我呀**— 一句话：**约球、分队、记分，一个小程序搞定。**
> 本文是给设计出原型 + 后续工程落地的粗粒度方案。粒度原则：**流程/页面讲细，技术/算法讲粗。**
> 组织形式：单仓多工程、TS 全栈、Fastify+Prisma+MySQL、Taro 小程序、阿里云部署）

---

## 0. 它解决什么

业余羽毛球局子里反复出现的三个麻烦，对应三大功能：

| 痛点 | 现状（微信群里手动做） | 本产品 |
|---|---|---|
| **约球组局** | 群里接龙报名、数人头一团乱 | 一键建局 → 分享卡 → 自动报名/候补/签到 |
| **双打怎么分** | 现场靠喊、水平不均、老和同一拨人打 | 选人 → 选模式 → 自动分队/轮转 → 可拖拽微调 |
| **谁赢了/谁强** | 没人记，打完就忘 | 每局计分 → 今日榜 + 个人累计战绩 |

---

## 1. 目标用户 & 场景

- **组织者（局长）**：发起一场球、拉人、现场排兵、记分。
- **球友**：点链接报名、看自己今天打几场、看战绩。
- **典型场景**：周末 8~16 人、2~4 片场地，双打为主。

---

## 2. 产品范围（v1 边界 + 红线）

**v1 做：**
- 建局 / 分享 / 报名 / 候补 / 签到
- 双打分组：**智能平衡** + **自动轮转** 两种模式（均支持手动拖拽微调）
- 每局计分 + 单场今日榜 + 个人累计战绩（简单积分/胜率）

**v1 不做（红线 / 后置）：**
- 🔴 **不碰钱**
- 🔴 **不做俱乐部/球队**：纯活动模型，每个局子独立，靠分享链接拉人；不做成员管理、入队审批。
- 跨局排行榜（圈子榜）、订阅消息提醒、淘汰赛/循环赛赛制、Web 端 → **v1.5 / v2**。

**通用硬约束：**

- 时区：DB 存 UTC、展示 `Asia/Shanghai`。
- 配置走 `config.yml`，不读环境变量；线上任何动作先经完整线下验证。

## 3. 核心对象与关系（轻量数据模型）

> 注意：「纯活动无球队」≠ 没有用户。用户仍按**微信 openid** 持久存在（有昵称/头像/默认水平/累计战绩），只是**没有"球队/俱乐部"这一层**。

```
球友 User ──发起──▶ 活动 Activity ──┬─ 报名 Signup（报名/候补/签到/请假）
   │(openid持久)                    │
   │                               └─ 分组 Round/Court ──▶ 对局 Match（含比分/胜负）
   └────────── 累计战绩（由 Match 汇总）◀───────────────────┘
```

| 对象 | 关键字段（粗） | 说明 |
|---|---|---|
| **User 球友** | openid、昵称、头像、性别(可选)、默认水平 | 微信登录持久身份；默认水平用于"智能平衡"，可记忆上次填的 |
| **Activity 活动/局** | 局长、标题、时间、场馆、场地数、人数上限、报名截止、默认分组模式、状态、备注 | 状态：`报名中 → 进行中 → 已结束 / 已取消` |
| **Signup 报名** | 活动、球友(或临时球友)、状态、本场水平、+1 | 满员自动进候补；有人退自动补位；本场水平默认取 User 默认值，局长可改 |
| **Round / Match 轮次/对局** | 轮次、场地号、A队、B队、比分、胜负、状态 | 队制或个人轮转两种产出共用同一结构 |
| **临时球友 Guest** | 仅昵称、无 openid | 现场常有人没扫码，做占位身份（参考 camping 的 actor 模型） |

## 4. 端到端主流程（一条线）

```
局长建局 ──▶ 生成分享卡 ──▶ 球友点链接报名/候补
                                   │
                              到场签到（局长勾 / 自助）
                                   │
                          选人 + 选模式 ──▶ 自动分队/轮转 ──▶ 拖拽微调 ──▶ 开打
                                   │
                          逐局/逐轮计分 ──▶ 实时对阵看板
                                   │
                          收尾结算：今日榜 + 分享战报
                                   │
                          沉淀到每个人的个人累计战绩
```

## 5. 功能模块详解

### 5.1 报名组局

- **建局**：标题、时间、场馆/地点、场地数、人数上限、报名截止、默认分组模式、备注。
- **报名**：报名 / 取消 / 候补（满员自动候补，退出自动补位）/ 可带朋友(+1) / 请假。
- **签到**：当天实际出勤（局长勾选或球友自助）；签到名单 = 分组的输入。可临时添加无微信的"临时球友"。

### 5.2 双打分队分组 ★ 差异化核心（v1 双模式，均可手动拖拽）

发起分组时选模式：

**A. 智能平衡分队（Balanced）** — 想每场势均力敌
- 输入：参赛者 + 各自水平 + 场地数。
- 产出：把人配成两两组合 / 分两队，使每片场地双方实力尽量接近。
- 可选：混双约束（按性别搭配）。
- 适合：水平参差、想打得公平。

**B. 自动轮转（Rotation：美式 / 墨式）** — 想多认识人、个人积分赛
- **美式 Americano**：每轮系统换搭档/对手，尽量不重复，个人积分累计。
- **墨式 Mexicano**：按当前积分动态配对（强弱搭），更平衡刺激。
- 产出：多轮赛程，每轮各场地对阵 + 谁轮空（出场次数/休息均衡）。
- 适合：散客局、社交局、个人天梯感。

> 引擎目标（共通）：出场次数均衡、尽量不重复搭档/对手、休息均衡、可选水平/混双约束。**任何模式都允许局长手动拖拽换人**——算法给草稿，人拍最终板。

### 5.3 记分排名

- **计分**：每局快速录入（大按钮 +1/−1 或直接填比分），定胜负，支持改判。
- **单场今日榜**（随模式不同）：
  - 队制（平衡模式）：胜场 / 胜率 / 净胜分。
  - 个人制（轮转模式）：个人累计积分。
  - 评本场 MVP。
- **个人累计战绩**（跨局，按 openid 汇总）：参赛局数、胜率、积分、最佳搭档、苦主、趋势图。
- **跨人累计榜**：v1 暂只做"个人战绩页"；公共排行榜需要一个范围（无球队时不好定边界），放到 v1.5 用轻量「圈子码」实现。

## 6. 页面清单 & 每页要素 ★ 给设计师的重点

> 约 10 个业务页 + 2 张分享卡，规模与 camping 小程序相当。

| # | 页面 | 目的 | 关键要素 / 组件 | 主要操作 |
|---|---|---|---|---|
| 1 | **首页 / 我的局** | 入口聚合 | Tab：报名中 / 进行中 / 已结束；活动卡（时间·地点·已报/上限·状态）；空态引导 | 发起新局、进入详情、粘贴链接加入 |
| 2 | **发起活动** | 建局表单 | 标题、时间选择器、场馆、场地数、人数上限、报名截止、默认分组模式、备注 | 提交 → 生成分享卡 |
| 3 | **活动详情**（枢纽页） | 看信息 + 报名 + 分享 | 信息区（时间/地点）、报名名单头像墙（已报名/候补/请假）、我的操作区、局长操作区、分享按钮 | 报名/取消/+1、签到、开始分组、编辑、取消活动、分享 |
| 4 | **签到 / 出勤** | 确定参赛名单 | 签到清单（勾实到）、候补补位、添加临时球友、各人本场水平确认 | 勾签到、补位、改水平、加临时球友 |
| 5 | **分组排兵** | 生成对阵 | ① 选人 → ② 选模式（平衡/轮转）→ ③ 设置（场地数·轮数·混双开关）→ ④ 看板（场地×轮次）→ ⑤ 拖拽微调 | 一键生成、拖拽换人、确认开打 |
| 6 | **对阵看板** | 比赛进行中 | 各场地当前对局卡（A队 vs B队）、本轮/下一轮、轮空提示、（可选）计时 | 点击进入计分、切换轮次 |
| 7 | **计分录入** | 记一局结果 | 大字比分、+1/−1 按钮、胜负标识、确认/改判 | 记分、确认、改判 |
| 8 | **本场结算** | 收尾 | 今日榜（队制或个人积分）、MVP、分享战报 | 查看榜单、分享战报 |
| 9 | **个人战绩** | 沉淀荣誉 | 累计局数/胜率/积分、最佳搭档、苦主、趋势图（自己可看，他人只读） | 查看、分享 |
| 10 | **我的 / 设置** | 个人中心 | 资料（昵称头像/默认水平/性别）、我的报名、帮助、关于 | 编辑资料、查历史 |

**分享卡（设计资产，非页面）：**
- **活动分享卡**：标题 + 时间地点 + 已报/上限 + "点我报名"，群里转发拉人（小程序增长主链路）。
- **战报分享卡**：今日榜 + MVP，打完晒成绩。

## 7. 里程碑分期

| 阶段 | 内容 |
|---|---|
| **v1 (MVP)** | 建局→分享→报名/候补→签到→分组(平衡+轮转, 可拖拽)→计分→今日榜；个人战绩基础版 |
| **v1.5** | 跨局「圈子码」排行榜、订阅消息提醒（开赛/截止）、混双优化、候补/请假流程完善、更多赛制（循环赛/淘汰赛）、战绩可视化增强 |
